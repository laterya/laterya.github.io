# 关于这个站点

`Laterya Notes` 是一个个人博客，用来记录三类内容：

- 工程实践：那些在项目里真正复用得上的方法、结构和工作流。
- 产品观察：功能背后的取舍、信息架构和体验判断。
- 长期项目：正在推进的事情，以及它们在时间里的变化轨迹。

我希望它不是一个只追求更新频率的内容仓库，而是一个可以持续回看的外部记忆系统。

## 技术栈

- [Astro](https://astro.build/)
- [Firefly](https://github.com/CuteLeaf/Firefly) 主题
- GitHub Pages + GitHub Actions
- Markdown / MDX 内容工作流

## 为什么用 Firefly

我想要的不是纯粹极简到没有辨识度的博客，也不是过重的全栈系统。

Firefly 刚好在中间：

- 视觉完成度比官方 starter 高很多
- 内容组织和写作工作流仍然保持清晰
- Astro 静态生成让部署和维护成本都比较低
